# Product Specs Skill

## Activation

Auto-fires when the user describes a feature, asks "what should we build?",
or creates a new `specs/` directory.

## PRD template

```markdown
# PRD: {Feature Name}

**Status**: draft | review | approved
**Owner**: jade-vp-product
**Target date**: YYYY-MM-DD

## Problem
[What customer pain does this solve?]

## Customer Persona
[Who is the primary user?]

## Success Metrics
| Metric | Baseline | Target |
|--------|----------|--------|

## Scope
**In scope**: ...
**Out of scope**: ...

## User Stories
- As a [persona], I want to [action], so that [value].

## Acceptance Criteria
- [ ] ...
```
